Circuit 1 - Multiplexer as a switch:

A multiplexer (MUX) is basically a switch which connects one of 2nMUX data inputs to a single output. The n select inputs of the MUX determine which of the MUX data inputs will appear at the MUX output. Often a MUX also has an enable input. If the MUX is not enabled, none of the data inputs appear on the MUX output. The first circuit you build will select one of four numbers to be displayed.

You will use the eight data switches of the breadboard to make four two-bit numbers. The first will be SW1, SW2 and the fourth will be SW7, SW8. Two 4:1 MUXes will be used to select which two-bit number appears at the MUX outputs.

Circuit 2 - Parity generator:

A multiplexer can be used as a standard building block to implement any desired logic function. For a function with n inputs, if the number of selects is at least (n-1) this implementation does not require any minimization techniques. Further, if the specification of the logic function changes, the circuit can be modified by simply changing the MUX data inputs corresponding to the changed truth table values.

Consider a circuit to be used for binary information  transmission using three bits D2, D1 and D0. Here D2 is the most significant data bit. In order to be able to detect single bit errors in transmission, an extra bit P is to be added to the binary message to make the number of ones odd in the transmitted output. This output P is a function of 3 variables and can be created with a 4:1 MUX.
